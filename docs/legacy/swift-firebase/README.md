# Swift/Firebase Legacy Archive

These files were copied from the original Pluckr SwiftUI + Firebase repository.

They are here for reference, not because v2 still uses that stack.

## Why Keep Them

- they capture the original product thinking
- they document old flows, naming, and constraints
- they preserve architecture and design context while we rebuild

## What To Trust Them For

Good reference:

- product scope
- feature ideas
- design direction
- prior lessons and failure modes

Not source of truth for v2:

- Firebase-specific setup
- Swift-specific structure
- old deployment assumptions

## Folder Layout

- `root/`: top-level project docs from the original repo
- `dev-guidelines/`: architecture, cleanup, security, and design notes
- `scripts/`: Firebase-era operational docs
